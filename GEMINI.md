# Project Instructions: Rapid Prototype Interview

This project is a 3-hour rapid prototyping challenge for a real-world client scenario. All actions must prioritize speed, professional quality, and shipping a functional MVP.

## Sub-Agent Orchestration Strategy

To maximize efficiency within the 180-minute window, follow these delegation rules:

### 1. Scoping & Strategy (Minutes 0-15)
- **Agent:** `@product-manager`
- **Trigger:** Use immediately upon receiving the client scenario.
- **Goal:** Identify the core value proposition, define the MVP boundary, and explicitly list what features to CUT to ensure delivery.

### 2. Visual Roadmap (Minutes 15-20)
- **Task:** Use Excalidraw (AI Text-to-Diagram) to visualize the MVP flow.
- **Goal:** Create a 1-page visual guide to keep the build on track and use as the opening slide for the demo.

### 3. Implementation (Minutes 20-120)
- **Agent:** `@react-specialist`
- **Trigger:** Use for generating complex UI components, server actions, and layout structures.
- **Constraint:** Always instruct the agent to use **Shadcn UI** and **Tailwind CSS**.
- **Mandate:** Maintain high-quality code standards and responsive design by default.

### 3. Quality & Validation (Minutes 120-140)
- **Agent:** `@qa-expert`
- **Trigger:** Use after the core features are functional.
- **Goal:** Generate critical validation tests (unit or integration) for the most sensitive logic to demonstrate "Product Thinking" and engineering robustness.

### 4. Crisis Management (Anytime)
- **Agent:** `@debugger`
- **Trigger:** Use if any error or environment issue persists for more than 5 minutes.
- **Goal:** Resolve blockers instantly to maintain momentum.

### 5. Final Polish & Communication (Minutes 150-180)
- **Agent:** `@documentation-engineer`
- **Trigger:** Use for the final 30 minutes.
- **Goal:** Generate a professional README.md that highlights:
  - Technical choices (Next.js, Shadcn, TypeScript).
  - Scope decisions (What was built vs. what was cut).
  - Quality assurance (Tests generated by `@qa-expert`).
  - Clear value proposition for the client.

## Core Mandates
- **Speed & Utility:** Prioritize working features over aesthetic perfection.
- **Accessibility:** Use semantic HTML and established UI patterns.
- **Tone:** Professional and aligned with the specific client scenario provided.
- **Quality:** Ensure the demo-ready paths are robust and error-free.
